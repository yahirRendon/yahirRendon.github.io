Conway`s Game of Life is a classic cellular automaton developed in 1970 by John Horton Conway. It consists of a two-dimensional grid of cells. Individual cells can either be alive or dead to begin with and can be set by either a user or automatically. After the initial state of cells is set the program begins evolving via steps in time to update the cell states by following four simple rules. Any cell in the alive state with fewer than two living neighbors dies, any cell in the alive state with two or three living neighbors lives into the next generation, any cell in the alive state with more than three living neighbors dies, and any cell in the dead state with exactly three living neighbors becomes a living cell in the next generation. These rules serve to simulate real life ecosystems reflecting underpopulation, homeostasis, overpopulation, and reproduction. From these simple rules incredibly complex patterns can arise making the game capable of endless possibilities.
My version follows the same basic rules with a minor twist in the board layout. The original version uses a two-dimensional orthogonal grid of square cells while mine uses circles aligned in near optimal circle packing manner. This offsets every other row to fit more circles in the same space over an orthogonal layout. Additionally, the grid layout is stripped of all unnecessary lines to create an extremely minimal appearance. To aid in cell selection in the static state the grid of cells is revealed around the mouse cursor. All the basic patters and designs are still possible but will appear different due to this layout. Finally, individual cells recall the amount of time spent in the alive state and a heatmap is generated with this data. Learn more about the Game of Life here.
