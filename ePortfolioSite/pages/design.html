Software Design and Engineering
A modern and responsive weather data dashboard
+
Description

The artifact I chose to use to demonstrate my software design and engineering abilities is the weather station project from CS-350, Emerging Technologies. This project was created in December 2020 as a columniation of work in the final milestone. It consisted of using a RaspberryPi and the GroovePi HAT/sensors to collect weather data. A JSON file is updated at 30-minute intervals during daylight hours only. The specific data collected consists of temperature, humidity, light sensitivity, and dates. Each reading and the data collected is then displayed in a local web dashboard.

Justification

I chose this project as an artifact for my ePortfolio because of the potential for improved complexity. This project demonstrate my ability to work with an embedded system, sensors, work with JSON, write a script in Python, write HTML, CSS, and JavaScript. It was also an opportunity to further work on a project that reflects my goal of working not only with code but with sensors and devices that interact with the world around us. This project had a solid foundation for collecting data but the displaying of that data in a meaningful way was greatly lacking. This presented a great opportunity to showcase my design and engineering skills to deliver a visually appealing and user driven web dashboard.

Reflection

This project posed various challenges that all expanded my knowledge on several aspects of web development. First, creating charts and graphs on the web required me to learn how to use the JavaScript library CanvasJS. This was not something that was addressed in the original course, so I had to spend time reviewing the CanvasJS documentation to understand how to structure my data and render the desired charts that I wanted. This ability to learn a new library by reviewing documentation is an incredibly important skill to have. It is definitely a skill on to itself to be able to take a developerâ€™s notes and incorporate their framework successfully into your own project.

Another challenge, that required forethought and intention was the layout and design of the web dashboard. It was important for the dashboard to be responsive as it is important to account for the variety of screen sizes people use in the world. I also wanted it to be a reusable template that can be populated with data from different cities. This led me to explore different ways of using HTML and CSS such to meet the needs of the current enhancements but future growth as well. To test my design and engineering approach I created data for two additional cities to see how scalable my project was. It was challenging to develop a solution that displayed dashboard data but also allowed users to change cities and learn more about the project. However, by session storage and JavaScript functions I was able to write a scalable solution.

In the end, this project was greatly improved in its design and engineering by heavily working on the front-end of the web dashboard. The prior HTML file contained JavaScript and CSS has been properly divided into individual files that reference each other. A modern and appealing website layout has been created using CSS grid. My code is written using best practices for each language utilized, it is appropriately commented for communicating intentions, creative solutions are demonstrated in costume functions, and overall employs modular programming concepts that will allow the project to scale as necessary through thoughtful initial design.
